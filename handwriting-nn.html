The Goal: An Autograder

Some of the most tedious work I do as a teaching assistant is grading. While it is important to take time to manually review students' work to give feedback, grading short answer questions for points always feels like a necessary evil. But it shouldn’t have to be. It should be possible to build something that can automate this.

Deep Learning About Deep Learning

The very first step in building something like this is to create a system that can recognize handwriting (the student work I grade is, after all, handwritten). Instead of relying on a deep learning library to accomplish this, I wanted to take this opportunity to really understand the inner workings of neural networks.

I hadn’t yet taken a course on deep learning, but fortunately, there are some truly excellent resources available online to learn about the beauty of neural networks. I primarily used Michael Nielsen’s book Neural Networks and Deep Learning along with outstanding visualizations and explanations by 3Blue1Brown.

The neural network I created was a feedforward network, using stochastic gradient descent with backpropagation, all implemented using only Numpy. Training on 60,000 handwritten digits in the MNIST dataset, this network achieved an accuracy of 95.30%.

A sample of the MNIST dataset containing 28x28 pixel images of handwritten digits

PyTorch

Knowing how to create neural networks from scratch, I could safely abstract away some of the complexities. So, I implemented the same network — using stochastic gradient descent and the quadratic cost function — in PyTorch. The behind-the-scenes optimizations of PyTorch improved the accuracy to 96.05%. A little better.

To further improve the network, I swapped the quadratic cost function with the cross-entropy cost function to mitigate neuron saturation. This bumped the accuracy up to 97.71%. Finally, I added L2 regularization to reduce overfitting, which further improved the accuracy. The final accuracy was 98.06%, meaning the network correctly classified 98,060 of the 10,000 test images. Not bad!
